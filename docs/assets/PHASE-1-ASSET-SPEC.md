# Phase 1 Asset Specification
Project: Motorcade (motorcade.vip)

Scope: Public Website + Baseline UI Shell  
Status: Draft for Approval  
Phase: 1 (Website Assets Only)

This document defines the REQUIRED assets for Phase 1 and the
PSYCHOLOGICAL INTENT behind each asset class.

This is a CONTRACT document.
Assets generated outside this specification are considered non-canonical.

-----------------------------------------------------------------------

## Purpose of Phase 1 Assets

Phase 1 assets exist to establish:
- Trust on first contact
- Professional credibility
- Calm authority
- Visual restraint

They are NOT intended to:
- Explain operations
- Demonstrate scale
- Show personnel
- Represent internal workflows

Phase 1 assets are external-facing only.

-----------------------------------------------------------------------

## Global Asset Rules (Non-Negotiable)

Applies to ALL Phase 1 assets:

- No human faces
- No guards or uniforms
- No weapons or tactical imagery
- No stock-photo look
- No cinematic or “action” styling
- No exaggerated contrast or HDR
- No bright alert colors (red/orange as dominant tones)

Primary emotional tone:
CALM · CONTROLLED · PROFESSIONAL · TRUSTWORTHY

-----------------------------------------------------------------------

## Asset Categories

### 1. Homepage Hero Background

Purpose:
- Establish first-impression credibility
- Signal seriousness without intimidation

Psychological intent:
- “This company reduces chaos”
- “This is not a marketing gimmick”

Visual characteristics:
- Subtle gradient or real-world abstract texture
- Dark neutral palette (navy, slate, charcoal)
- No identifiable people, vehicles, or weapons
- Soft lighting, low noise

Technical requirements:
- Aspect ratio: 16:9
- Formats: WebP (primary), PNG/JPG fallback
- Resolutions: 2560px, 1440px, 1024px, 768px widths

-----------------------------------------------------------------------

### 2. Section Backgrounds (Website)

Sections covered:
- Services
- Trust / Compliance
- Call-to-Action
- Footer

Purpose:
- Visual separation without distraction
- Rhythm and hierarchy

Psychological intent:
- Stability
- Order
- Predictability

Visual characteristics:
- Lighter variants of hero tones
- Soft gradients or extremely subtle textures
- No imagery suggesting action or urgency

Technical requirements:
- Seamless / repeat-safe where applicable
- WebP + PNG/JPG fallback
- Optimized for performance

-----------------------------------------------------------------------

### 3. UI Icons (Website + Baseline UI Shell)

Purpose:
- Rapid comprehension
- Reduce cognitive load

Psychological intent:
- Clarity over personality
- Function over decoration

Icon style rules:
- Monochrome or duotone
- Consistent stroke weight
- Geometric, symmetrical
- No playful curves
- No novelty metaphors

Required icon concepts (minimum):
- Shield / Protection
- Verification / Check
- Location / Map
- Time / Clock
- User / Identity (generic)
- System / Settings
- Alert (non-alarmist)

Formats:
- SVG (primary)
- PNG fallback (1x, 2x)

-----------------------------------------------------------------------

### 4. Trust & Compliance Badges

Purpose:
- Reinforce legitimacy
- Provide visual anchors for trust claims

Psychological intent:
- Verification
- Accountability
- Order

Visual characteristics:
- Shielded or rectangular forms
- Muted colors
- No seals, ribbons, or “award” styling

Example badge concepts:
- Licensed
- Insured
- Background Checked
- 24/7 Dispatch
- Vetted Personnel

Formats:
- SVG + PNG
- Scalable without pixelation

-----------------------------------------------------------------------

### 5. Dividers & Structural Elements

Purpose:
- Reduce cognitive load
- Improve scanability

Psychological intent:
- Structure
- Calm segmentation

Visual characteristics:
- Thin lines
- Soft contrast
- Or spacing-based separation
- Optional ultra-subtle noise texture

Explicit exclusions:
- Heavy borders
- Boxes around everything
- Decorative flourishes

Formats:
- SVG (preferred)
- PNG for textures

-----------------------------------------------------------------------

### 6. Baseline UI Shell Visuals (Non-Role-Specific)

Purpose:
- Provide neutral UI framing
- Support login/auth shell without implying workflows

Psychological intent:
- Seriousness
- Enterprise-grade restraint

Includes:
- Login background
- Auth container background
- Neutral UI gradients/textures

Explicit exclusions:
- Ops dashboards
- Employee workflows
- Status indicators
- Data visualizations

Formats:
- WebP + PNG fallback
- Responsive-safe

-----------------------------------------------------------------------

## Explicit Exclusions (Phase 1)

Phase 1 MUST NOT include:
- Ops portal visuals
- Employee portal visuals
- Guard imagery
- Vehicles or convoy scenes
- Action scenarios
- Workflow diagrams
- Screenshots of unfinished UI

Those belong to later phases by design.

-----------------------------------------------------------------------

## Acceptance Criteria (Phase 1 Complete)

Phase 1 assets are considered complete when:

- All required categories exist
- Assets comply with psychological intent
- No exclusions are violated
- Assets are consistent in tone and style
- Assets can be regenerated without loss of intent

Only after acceptance may:
- assets-manifest.json be finalized
- assets be considered canonical
- playbooks 15–17 be resumed

-----------------------------------------------------------------------

## Authority

This specification is authoritative for Phase 1.

If an asset conflicts with this document,
THE DOCUMENT WINS.
